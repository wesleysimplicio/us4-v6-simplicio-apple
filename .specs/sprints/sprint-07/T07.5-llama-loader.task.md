# T07.5 - Deepen llama loader coverage

## Context

The current Llama path already supports fixture manifests and a toy GGUF route,
but Sprint 07 still expects a loader contract that feels closer to a real
Llama ingestion surface.

## Current progress

- Binary Llama assets already route by family/model without explicit `--model`
- The next slice is sibling-manifest hydration for GGUF/safetensors so typed
  config metadata and tokenizer-oriented hints stay visible to the adapter path

## Acceptance criteria

- [ ] GGUF, safetensors, and tokenizer-oriented metadata are represented in the
      Llama loader contract
- [ ] Family/model routing stays deterministic when assets omit explicit CLI model input
- [ ] Native tests protect the visible loader telemetry

## Validation

- build/lint/unit
- native CLI E2E
- loader contract regression
